### User Story: Централизация текстов через i18n

**ID:** `008-i18n-text-storage`
**Приоритет:** Medium
**Статус:** Draft

---

## User Story

Как **разработчик**,
я хочу **хранить все тексты бота централизованно через i18n-слой**,
чтобы **убрать строки из кода, упростить поддержку и подготовить систему к редактированию текстов и мультиязычности**.

---

## Контекст

Сейчас тексты:

* захардкожены в `handlers`, `utils`, `dataProvider`
* дублируются
* не типизированы
* не могут редактироваться без деплоя

При этом уже есть `messagesController`, что позволяет рассматривать тексты как данные.

---

## Scope

### Входит

* Введение функции `t(key)`
* Вынесение текстов в отдельный слой (`src/i18n`)
* Типизация ключей сообщений
* Замена строк в коде на `t(key)`
* Поддержка fallback (код → БД в будущем или наоборот)

### Не входит

* Полноценная мультиязычность (несколько языков)
* pluralization и форматирование
* интеграция сторонних i18n-библиотек

---

## Acceptance Criteria

### 1. Централизованный доступ к текстам

* В проекте есть функция:

```ts
t(key: MessageKey): string
```

* Все пользовательские тексты в handlers и utils получаются только через `t()`

---

### 2. Хранение текстов в коде

Создана структура:

```
src/i18n/
├── messages.ts
├── index.ts
└── types.ts
```

Пример:

```ts
export const messages = {
  welcome: {
    message: 'Добро пожаловать',
  },
} as const;
```

---

### 3. Типизация ключей

* Все ключи выводятся автоматически:

```ts
type MessageKey = DeepKeys<typeof messages>;
```

* Вызов `t('invalid.key')` вызывает ошибку компиляции

---

### 4. Замена строк в коде

До:

```ts
ctx.reply('Добро пожаловать')
```

После:

```ts
ctx.reply(t('welcome.message'))
```

---

### 5. Fallback-логика

Функция `t()`:

* возвращает текст из локального словаря
* при отсутствии ключа:

  * логирует ошибку
  * возвращает сам ключ (или safe fallback)

---

### 6. Готовность к БД

Архитектура позволяет:

* в будущем подключить `messages` из БД
* переопределять значения без изменения кода

(реализация загрузки из БД не обязательна в рамках этой задачи)

---

## Технические детали

### Пример реализации

```ts
const messages = { ... } as const;

export function t(key: MessageKey): string {
  const value = get(messages, key);

  if (!value) {
    console.error(`Missing i18n key: ${key}`);
    return key;
  }

  return value;
}
```

## Definition of Done

* [ ] Создан модуль `i18n`
* [ ] Все тексты вынесены из handlers
* [ ] Используется `t()` вместо строк
* [ ] Ключи типизированы
* [ ] Проект компилируется без ошибок
* [ ] Бот работает как раньше (/start, навигация)

---

## Риски

* Пропущенные строки → частично остаётся старый подход
* Ошибки в ключах → решается типизацией
* Слишком раннее усложнение → ограничено простым слоем без библиотек

* без внешних зависимостей
* с полной типизацией
