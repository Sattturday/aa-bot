### User Story: Централизованная обработка ошибок в хендлерах

**Title:** Централизовать обработку ошибок в Telegram-хендлерах

**As a** пользователь Telegram-бота
**I want** получать понятное сообщение при ошибке
**So that** я понимаю, что произошло и что делать дальше

---

### Контекст

В текущей реализации каждый хендлер содержит собственный `try-catch` с `console.error`.
Это приводит к:

* дублированию кода
* отсутствию единообразного поведения при ошибках
* отсутствию feedback пользователю

---

### Описание решения

Ввести универсальную обёртку `withErrorHandler`, которая:

* централизует логирование ошибок
* отправляет пользователю fallback-сообщение
* используется через фабрику регистрации хендлеров

---

### Acceptance Criteria

1. Все хендлеры больше не содержат локальные `try-catch`
2. Ошибки логируются через единый механизм (`console.error` с label)
3. При любой ошибке пользователь получает сообщение:

   * `"Произошла ошибка. Попробуйте ещё раз или нажмите /start"`
4. Ошибка при `ctx.reply` не ломает выполнение (игнорируется)
5. Обёртка применяется ко всем хендлерам через фабрику (`registerCategory` и другие)
6. Поведение бота не меняется в случае отсутствия ошибок

---

### Технические задачи

* Создать файл:

  * `src/utils/handlers/withErrorHandler.ts`
* Реализовать функцию-обёртку `withErrorHandler(handler, label)`
* Интегрировать обёртку:

  * в `registerCategory`
  * в остальные хендлеры (`startHandler`, `backHandler`, `adminHandler`, и т.д.)
* Удалить дублирующиеся `try-catch` из существующих хендлеров

---

### Definition of Done

* Код компилируется без ошибок
* Все хендлеры используют `withErrorHandler`
* В коде отсутствуют дублирующиеся `try-catch` в хендлерах
* При искусственно вызванной ошибке:

  * ошибка логируется
  * пользователь получает сообщение

---

### Дополнительно (необязательно)

* Добавить разные сообщения для разных типов ошибок (например, network / validation)
* Подключить внешний логгер (например, Sentry) вместо `console.error`

---

Если кратко: убираем хаос из `try-catch`, делаем одну точку контроля и нормальный UX при падениях.
