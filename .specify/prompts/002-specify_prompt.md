# 002-refactor-handlers

## User Story 1: Рефакторинг обработчиков Telegram

**Как разработчик проекта**, я хочу разбить монолитный файл `handlers.ts` на модульную структуру с фабрикой регистрации обработчиков, чтобы уменьшить дублирование, упростить поддержку и сделать код предсказуемым для расширения.

---

### Обоснование:

- Текущий файл `src/utils/handlers.ts` содержит ~300+ строк и несколько повторяющихся блоков регистрации обработчиков.
- Повторяющаяся логика усложняет внесение изменений и повышает риск ошибок.
- Отсутствует явное разделение ответственности между категориями обработчиков.
- Нарушается принцип минимальной сложности — код выглядит проще, чем есть на самом деле.
- Рефакторинг позволит:
  - централизовать повторяющийся паттерн регистрации
  - уменьшить связность
  - упростить добавление новых категорий

---

### Acceptance Criteria:

1. **Фабрика регистрации обработчиков**
   - Создана функция `registerCategory`, инкапсулирующая повторяющийся паттерн `bot.action`.
   - Фабрика принимает:
     - экземпляр бота
     - категорию (`NavigationState`)
     - список ключей
     - функцию маппинга ключа (`KeyMapper`)
   - Вся логика:
     - `answerCbQuery`
     - получение пользователя
     - работа со стеком (`pushToStack`)
     - трекинг (`track`)
     - выбор обработчика (с изображением / без)
     — находится внутри фабрики.

2. **Удаление дублирования**
   - Повторяющиеся блоки регистрации обработчиков удалены из исходного файла.
   - Регистрация категорий выполняется через вызовы `registerCategory`.

3. **Разделение на модули**
   - Обработчики разбиты на отдельные файлы по ответственности:
     - стартовый обработчик (`/start`)
     - обработчики групп
     - обработчик "назад"
     - обработчик админки
     - fallback-обработчики
   - Каждый файл содержит только свою категорию логики.

4. **Оркестрация**
   - Создан единый файл `index.ts` в директории `handlers/`, который:
     - импортирует все обработчики
     - регистрирует их в одном месте (`registerAllHandlers`)

5. **Удаление старого файла**
   - Файл `src/utils/handlers.ts` удалён.
   - Все импорты обновлены на новую структуру.

6. **Совместимость поведения**
   - Поведение бота НЕ изменилось:
     - все кнопки работают как раньше
     - навигационный стек работает корректно
     - обработчики с изображениями продолжают работать

---

### Независимая проверка:

- Все кнопки Telegram:
  - вызывают соответствующие обработчики
  - корректно добавляют состояние в стек
- Команда `/start` работает как раньше
- Кнопка "Назад" возвращает на предыдущий экран
- Обработчики с изображениями отображают изображения
- Поведение fallback (неизвестные сообщения) не изменилось
- Код можно протестировать без знания внутренней реализации фабрики (через публичное поведение бота)

---

### UI:

- Не требуется
- Изменения касаются только внутренней структуры кода
