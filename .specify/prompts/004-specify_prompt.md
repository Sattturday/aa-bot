### User Story: Типизация кнопок и ключей

**Title:** Типобезопасные ключи экранов и кнопок

**As a** разработчик
**I want** иметь строгую типизацию `ScreenKey` и `ButtonKey`, выведенную из `buttonKeys`
**So that** исключить рассинхронизацию между `buttonKeys`, `buttons` и handlers и ловить ошибки на этапе компиляции

---

### Контекст

Сейчас ключи экранов и кнопок заданы как `string`. Это приводит к скрытым ошибкам:

* можно опечататься в ключе и не заметить
* нужно вручную синхронизировать несколько файлов
* ошибки проявляются только в рантайме

---

### Acceptance Criteria

1. **buttonKeys типизирован через `as const`**

   * все ключи становятся литеральными типами
   * добавлены типы:

     ```ts
     type ScreenKey
     type ButtonKey
     ```

2. **Тип `ScreenKey` используется как источник истины**

   * выводится как `keyof typeof buttonKeys`
   * используется во всех местах, где фигурируют экраны

3. **Тип `ButtonKey` корректно выводится**

   * как объединение всех значений массивов внутри `buttonKeys`
   * используется в обработчиках и утилитах

4. **buttons.ts использует строгую типизацию**

   * заменено:

     ```ts
     Record<string, ButtonRow[]>
     ```
   * на:

     ```ts
     Record<ScreenKey | ButtonKey, ButtonRow[]>
     ```

5. **utils.ts и handlers используют новые типы**

   * убраны `string` там, где можно использовать `ScreenKey` / `ButtonKey`
   * IDE подсказывает допустимые значения

6. **При добавлении нового экрана**

   * достаточно изменить только `buttonKeys`
   * TypeScript подсказывает, где нужно обновить код

7. **Компиляция падает при неконсистентности**

   * если ключ есть в `buttons`, но нет в `buttonKeys` — ошибка
   * если используется несуществующий ключ — ошибка

---

### Definition of Done

* Проект компилируется без ошибок
* Нет `string`-типов для ключей навигации
* Все связанные модули используют `ScreenKey` / `ButtonKey`
* Добавление нового ключа приводит к compile-time подсказкам, а не runtime-багам

---

### Технические заметки

* Основа решения — `as const` + indexed access types
* Избегать ручных union-типов — только вывод типов из объекта
* Не дублировать типы между файлами — импортировать из одного источника (`buttonKeys.ts`)
