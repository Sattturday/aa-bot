# Модель данных: Централизованная обработка ошибок в Telegram-хендлерах

## Объекты

### ErrorHandlerConfig

Конфигурация общей обёртки обработки ошибок.

| Поле | Тип | Описание |
|------|-----|----------|
| `label` | `string` | Человекочитаемый идентификатор сценария, используемый в логах |
| `fallbackMessage` | `string` | Текст, который отправляется пользователю при ошибке |
| `ignoreReplyFailure` | `boolean` | Флаг, разрешающий проглотить вторичную ошибку fallback-ответа |

### HandlerExecutionContext

Минимальный набор данных, извлекаемых из Telegram-контекста для диагностики.

| Поле | Тип | Описание |
|------|-----|----------|
| `userId` | `string \| null` | Telegram ID пользователя, если доступен |
| `chatId` | `string \| null` | Идентификатор чата, если доступен и нужен для отладки |
| `label` | `string` | Контекст текущего обработчика |

### LoggedHandlerError

Нормализованное представление сбоя, проходящее через единый путь логирования.

| Поле | Тип | Описание |
|------|-----|----------|
| `label` | `string` | Контекст обработчика |
| `error` | `unknown` | Исходная ошибка без преобразования в строку |
| `userId` | `string \| null` | Идентификатор пользователя |
| `stage` | `'handler' \| 'fallback'` | Стадия, на которой произошёл сбой |

## Состояния и переходы

### Successful Execution

1. Telegram вызывает зарегистрированный хендлер.
2. Wrapper передаёт управление исходной функции.
3. Исходная функция завершается без ошибок.
4. Дополнительных сообщений или логов wrapper не создаёт.

### Handler Failure

1. Telegram вызывает зарегистрированный хендлер.
2. Исходная функция выбрасывает исключение или отклоняет promise.
3. Wrapper пишет единый лог с `stage = 'handler'`.
4. Wrapper делает одну попытку отправить fallback-сообщение пользователю.
5. Выполнение завершается без повторного выброса ошибки наружу.

### Fallback Failure

1. Ошибка уже зафиксирована на стадии `handler`.
2. Попытка отправить fallback-сообщение завершается ошибкой.
3. Wrapper пишет второй лог с `stage = 'fallback'`.
4. Ошибка не пробрасывается и не инициирует новый цикл обработки.

## Затрагиваемые модули

| Модуль | Роль после изменения |
|--------|----------------------|
| `src/utils/handlers/withErrorHandler.ts` | Единая точка обработки ошибок |
| `src/handlers/factory.ts` | Подключает wrapper для фабричных `bot.action` |
| `src/handlers/start.ts` | Подключает wrapper для `/start` и стартовых action |
| `src/handlers/groups.ts` | Подключает wrapper для динамических и статических group-action |
| `src/handlers/back.ts` | Подключает wrapper для `back` |
| `src/handlers/admin.ts` | Подключает wrapper для `admin_panel` |
| `src/handlers/fallback.ts` | Подключает wrapper для `text` и `message` fallback |
| `tests/unit/utils/handlers/withErrorHandler.test.ts` | Проверяет единое поведение wrapper |
