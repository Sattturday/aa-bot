# Модель данных: Централизация текстов через i18n-слой

## Объекты

### MessageCatalog

Локальный словарь текстов, структурированный по ключам и используемый как дефолтный источник.

| Поле | Тип | Описание |
|------|-----|----------|
| `key` | `MessageKey` | Стабильный идентификатор сообщения |
| `value` | `string` | Текст сообщения на русском языке |
| `source` | `'catalog'` | Признак локального источника |

### RuntimeMessageMap

Карта сообщений, получаемая из runtime-данных (БД через data provider и кэш).

| Поле | Тип | Описание |
|------|-----|----------|
| `key` | `string` | Ключ из таблицы `messages` |
| `value` | `string` | Текущее значение, редактируемое через админ API |
| `source` | `'runtime'` | Признак runtime-источника |

### MessageKey

Типизированный ключ сообщения для обращения к `t(key)`.

| Поле | Тип | Описание |
|------|-----|----------|
| `path` | `string` | Иерархический ключ в формате `section.item` |
| `valid` | `boolean` | Принадлежность ключа разрешённому набору |

### FallbackEvent

Диагностическое событие при невозможности получить корректное текстовое значение.

| Поле | Тип | Описание |
|------|-----|----------|
| `key` | `string` | Искомый ключ |
| `reason` | `'missing' \| 'empty'` | Причина fallback |
| `resolvedFrom` | `'none' \| 'catalog' \| 'runtime'` | Источник результата |
| `timestamp` | `string` | Время события для отладки |

## Правила валидации

- Ключ, передаваемый в `t(key)`, должен принадлежать типизированному множеству `MessageKey`.
- Если runtime-значение пустое, оно считается отсутствующим и не должно подменять локальный дефолт.
- Локальный каталог обязан содержать значения для всех ключей, используемых в `handlers` и `utils`.
- Любое fallback-событие должно сопровождаться диагностической записью.

## Состояния и переходы резолвера

### Runtime Hit

1. `t(key)` получает ключ.
2. В runtime-карте найдено непустое значение.
3. Резолвер возвращает runtime-значение.

### Runtime Miss → Catalog Hit

1. `t(key)` получает ключ.
2. В runtime-карте нет значения или оно пустое.
3. В локальном каталоге найдено значение.
4. Резолвер возвращает локальный текст.

### Full Miss

1. `t(key)` получает ключ.
2. Значение отсутствует и в runtime-карте, и в локальном каталоге.
3. Резолвер логирует `FallbackEvent` с причиной `missing`.
4. Резолвер возвращает `key` как safe fallback.

## Затрагиваемые модули

| Модуль | Роль после изменения |
|--------|----------------------|
| `src/i18n/messages.ts` | Локальный каталог сообщений |
| `src/i18n/types.ts` | Типы ключей и служебные типы вывода |
| `src/i18n/index.ts` | Публичный резолвер `t(key)` |
| `src/db/dataProvider.ts` | Поставка runtime-данных для резолвера |
| `src/handlers/*` | Переход с литеральных строк на `t(key)` |
| `src/utils/utils.ts` | Переход вспомогательных сообщений на `t(key)` |
