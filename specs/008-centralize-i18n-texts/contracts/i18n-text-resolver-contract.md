# Контракт: I18N Text Resolver

## Назначение

Зафиксировать единый контракт получения пользовательского текста через `t(key)` для всех модулей бота.

## Интерфейс

```ts
t(key: MessageKey): string
```

## Входные условия

- `key` принадлежит типизированному множеству `MessageKey`.
- Ключ соответствует одному из элементов централизованного словаря.

## Правила разрешения значения

1. Резолвер проверяет runtime-значение для ключа (данные БД через текущий provider).
2. Если runtime-значение отсутствует или пустое, резолвер использует локальный каталог.
3. Если значение отсутствует в обоих источниках, резолвер:
   - пишет диагностическую запись с ключом и причиной;
   - возвращает `key` как safe fallback.

## Выходные гарантии

- Возвращаемое значение всегда строка.
- Резолвер не выбрасывает исключения в пользовательский поток из-за отсутствующего ключа.
- Повторяемый вызов с одинаковыми входными данными возвращает одинаковый результат.

## Нефункциональные требования

- Использование `t(key)` обязательно для пользовательских текстов в `handlers` и `utils`.
- Контракт не вводит внешних i18n-зависимостей.
- Контракт сохраняет совместимость с будущим переопределением источника текстов (например, расширение данных из БД).

## Сценарии проверки контракта

1. Корректный ключ с runtime-значением возвращает runtime-текст.
2. Корректный ключ без runtime-значения возвращает локальный текст.
3. Отсутствующий ключ возвращает сам ключ и порождает диагностическую запись.
