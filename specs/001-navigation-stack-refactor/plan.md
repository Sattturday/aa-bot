# План реализации: Рефакторинг навигационного стека

**Ветка**: `001-navigation-stack-refactor` | **Дата**: 2026-03-27 | **Спецификация**: [spec.md](./spec.md)
**Входные данные**: Спецификация фичи из `specs/001-navigation-stack-refactor/spec.md`

## Резюме

Вынести логику навигационного стека (`pushToStack`, `popFromStack`, `clearUserNavigationStack` и приватное состояние) из точки входа `src/index.ts` в отдельный модуль `src/utils/navigationStack.ts`. Обновить импорты в `src/utils/handlers.ts`. Функциональность остаётся идентичной — изменения сугубо структурные.

**Language/Version**: Node.js 18+, TypeScript (strict mode), компиляция через `tsc`
**Primary Dependencies**: Telegraf 4.x (Telegram Bot API)
**Storage**: Навигационный стек — in-memory, SQLite не затрагивается
**Testing**: Тестовый фреймворк не настроен (см. CLAUDE.md)
**Target Platform**: Linux server (один Docker-контейнер)
**Project Type**: Telegram-бот, однопроцессное Node.js-приложение
**Performance Goals**: Не применимо (структурный рефакторинг)
**Constraints**: Ограничение стека — 10 000 пользователей, порядок LIFO
**Scale/Scope**: 3 функции, 2 файла для обновления импортов, 1 новый файл
