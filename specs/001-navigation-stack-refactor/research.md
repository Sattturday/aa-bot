# Исследование: Рефакторинг навигационного стека

**Ветка**: `001-navigation-stack-refactor` | **Дата**: 2026-03-27

## Контекст

Задача тривиальна: перенести три функции (`pushToStack`, `popFromStack`, `clearUserNavigationStack`) и приватное состояние (`userNavigationStack`, `MAX_NAV_USERS`, `trimNavigationStack`) из `src/index.ts` в новый модуль `src/utils/navigationStack.ts`.

---

## Решение 1: Расположение нового модуля

**Решение**: `src/utils/navigationStack.ts`

**Обоснование**: Все утилиты бота (handlers, utils, history) уже живут в `src/utils/`. Размещение туда же обеспечивает консистентность. Альтернатива `src/navigation.ts` в корне `src/` рассматривалась, но `src/utils/` уже является каноническим местом для вспомогательных модулей.

**Рассмотренные альтернативы**:
- `src/navigation.ts` — отклонено, нарушило бы сложившееся соглашение о структуре
- `src/core/navigationStack.ts` — отклонено, создание директории `core/` излишне для одного файла

---

## Решение 2: Устранение циклической зависимости

**Текущее состояние**: `handlers.ts` импортирует из `'..'` (то есть из `src/index.ts`), что является нестандартным импортом точки входа.

**Решение**: После создания `src/utils/navigationStack.ts` изменить импорт в `handlers.ts` с `from '..'` на `from './navigationStack'`. Это устраняет несоответствие.

**Обоснование**: Импорт из точки входа — антипаттерн (point-of-entry import). Он работает в Node.js, но затрудняет тестирование и понимание зависимостей. Прямой импорт из нового модуля — стандартная практика.

---

## Решение 3: Удаление экспортов из index.ts

**Решение**: Удалить экспорты `pushToStack`, `popFromStack`, `clearUserNavigationStack` из `src/index.ts` полностью. `index.ts` не обязан что-либо экспортировать — он является точкой входа, а не библиотечным модулем.

**Обоснование**: Сохранение экспортов как реэкспортов из `index.ts` — излишнее усложнение (принцип V конституции запрещает «обёртки на будущее»).

---

## Итог: затронутые файлы

| Файл | Действие |
|---|---|
| `src/utils/navigationStack.ts` | Создать — перенести логику стека |
| `src/index.ts` | Удалить объявления стека и их экспорты; добавить импорт из нового модуля (если нужен в index.ts — не нужен) |
| `src/utils/handlers.ts` | Изменить импорт с `'..'` на `'./navigationStack'` |
