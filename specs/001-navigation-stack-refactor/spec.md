# Спецификация фичи: Рефакторинг навигационного стека

**Ветка**: `001-navigation-stack-refactor`
**Создано**: 2026-03-27
**Статус**: Черновик

## Пользовательские сценарии и тестирование *(обязательно)*

### История 1 — Разработчик импортирует функции стека из отдельного модуля (Приоритет: P1)

Разработчик, работающий с навигационной логикой бота, может импортировать `pushToStack`, `popFromStack` и `clearStack` из единственного, специализированного модуля — а не из точки входа приложения.

**Почему этот приоритет**: Это основная цель рефакторинга. Все остальные истории зависят от существования этого модуля.

**Независимый тест**: Можно полностью проверить, убедившись, что новый модуль экспортирует три функции и что импорты в потребляющих файлах указывают на новое расположение.

**Сценарии приёмки**:

1. **Дано** существует модуль навигационного стека, **Когда** разработчик импортирует навигационные функции, **Тогда** источник импорта — `utils/navigationStack` (не точка входа приложения)
2. **Дано** точка входа приложения (`index.ts`), **Когда** она проверяется, **Тогда** она не содержит логики управления стеком

---

### История 2 — Навигация бота работает без регрессий (Приоритет: P2)

Пользователь Telegram-бота перемещается по меню (начиная с `/start`, нажимая кнопки, нажимая «назад»), и вся навигация работает точно так же, как до рефакторинга.

**Почему этот приоритет**: Рефакторинг не должен нарушить существующую функциональность. Поведение с точки зрения пользователя — окончательный критерий приёмки.

**Независимый тест**: Можно проверить, запустив бота и вручную пройдя путь `/start → выбор группы → назад`, убедившись, что состояние стека сохраняется корректно.

**Сценарии приёмки**:

1. **Дано** бот запущен после рефакторинга, **Когда** пользователь отправляет `/start`, **Тогда** главное меню отображается как прежде
2. **Дано** пользователь находится на экране подменю, **Когда** он нажимает «назад», **Тогда** он возвращается на предыдущий экран
3. **Дано** пользователь перешёл через несколько экранов, **Когда** он нажимает «назад» несколько раз, **Тогда** вся история навигации разматывается корректно

## Требования *(обязательно)*

### Функциональные требования

- **FR-001**: Система ДОЛЖНА предоставить отдельный модуль навигационного стека, содержащий функции `pushToStack`, `popFromStack` и `clearStack`
- **FR-002**: Точка входа приложения НЕ ДОЛЖНА содержать логику управления стеком после рефакторинга
- **FR-003**: Модуль обработчиков действий ДОЛЖЕН импортировать функции стека из нового отдельного модуля
- **FR-004**: Поведение стека ДОЛЖНО остаться идентичным текущей реализации (ограничение в 10 000 пользователей, порядок LIFO)
- **FR-005**: Новая функциональность НЕ ДОЛЖНА добавляться — рефакторинг носит исключительно структурный характер

### Ключевые сущности

- **Модуль навигационного стека**: Инкапсулирует управление историей навигации пользователя; предоставляет операции push, pop и clear
- **Стек навигации пользователя**: Хранилище в памяти с ключом по идентификатору пользователя, содержащее упорядоченный список посещённых экранов

## Критерии успеха *(обязательно)*

### Измеримые результаты

- **SC-001**: Все операции со стеком (push, pop, clear) доступны из единственного отдельного модуля — 0 функций стека остаётся в точке входа
- **SC-002**: Навигационные сценарии бота (`/start → подменю → назад`) завершаются без ошибок после рефакторинга
- **SC-003**: Количество строк в точке входа приложения, связанных с управлением стеком, равно нулю
- **SC-004**: Поведение бота с точки зрения пользователя не изменилось — все экраны и навигация «назад» работают идентично до и после рефакторинга

## Допущения

- Рефакторинг является сугубо внутренним; изменения в базе данных, API или внешних сервисах не предполагаются
- Навигационный стек остаётся в памяти (требование персистентности не добавляется)
- Существующее ограничение в 10 000 пользователей и поведение автоматической обрезки сохраняются в неизменном виде
- Только `src/index.ts` и `src/utils/handlers.ts` требуют обновления импортов; никакие другие файлы не импортируют функции стека напрямую
